回溯树的计算量#

无论是集合问题、排列问题、子集问题，还是更复杂的 N 皇后，基本都是使用回溯树来解决的。这里以排列问题为例，说明一下回溯树的计算量有多大。

首先说结论，回溯树的总计算量为阶乘，即 O(n!)，这个在 N 皇后那一部分也提到了。

下图是一个对有3个元素的数组做全排列的回溯树，可以看出：

  • 不考虑根节点，回溯树的第一层有 3 个节点；
  • 回溯树的第二层有 3 * 2 = 6 个节点；
  • 回溯树的第三层有 3 * 2 * 1 = 6 个节点；

所以对于规则为 n 的回溯树，总的计算量为：

n + n(n-1) + n(n-1)(n-2) \cdots + n!

【上图中写的公式的倒数第二项是错误的，倒数第二项应该是 n * (n-1) * (n-2) * \cdots * 3 * 2，而不是 (n-1)!】
